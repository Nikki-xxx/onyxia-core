---

# Objetivo

Este documento define el estándar oficial para toda la documentación del proyecto Onyxia.

Todo documento creado dentro del proyecto deberá respetar esta estructura para garantizar consistencia, trazabilidad y facilidad de mantenimiento.

---

# Estructura mínima

Todo documento deberá contener, como mínimo:

- Título.
- Versión.
- Estado.
- Objetivo.
- Contenido.
- Historial de cambios.

---

# Estados permitidos

- Borrador
- En desarrollo
- Activo
- Obsoleto

---

# Versionado

Los documentos utilizarán el formato:

Mayor.Menor

Ejemplos:

1.0

1.1

1.2

2.0

---

# Regla General

La documentación siempre tendrá prioridad sobre la implementación.

Ninguna funcionalidad importante deberá desarrollarse sin su documentación correspondiente.
